## 三、没讨论过的卡点(全局扫出来的)

1. **仓库和线上脱节(宏佳侧)**:他今天的修改直接本地构建推服务器,GitHub 还停在 76152f8。谁接手都对不上线上代码。
2. **加好友结果回调没落库(宏佳侧)**:每次触达都有一条"mh回调 未识别类型",是秒回的"加好友请求已发送"回执,他的分发没接住。发送失败时可能漏报。
3. **面试官改了时间,候选人不知道**:HR 在表格里改一面时间,已触达的候选人收到的还是旧时间。反向联动完全缺失(详见第五节)。
4. **@提醒的边界**:提醒@的是"最后改记录的人",如果是服务自己改的行,@不到人,只能退化成报名字。真人操作没问题。
5. **轮询规模**:每30秒拉全表2000+条 × 4条规则。现在没问题,表到万级会慢。方案:按"最后修改时间"增量拉,或上飞书事件订阅(表格变更推送)。不急,记着。
6. **深澜体系和我们体系的口径**:AI-HR 表候选人有深澜的面评和结论,手动简历走我们的标准包评分。两把尺子并存,未来最好统一到岗位标准包(深澜那套也从知识库读)。
7. **面评生成要 30-40 秒**:链路B里是同步生成,挡着字段回填。可改成先回填字段、面评后台补。体验问题,不是功能问题。
8. **候选人重复投递**:同一人换个岗位再投,现在会当新人处理。要不要按手机号识别"回头客"、把历史面评带出来?值得讨论。

## 四、功能地图(可以加什么)

按业务流分三层,每条都标了依赖。

### A. 对话层(候选人体验)
- **知识库飞书化(推荐尽快做)**:现在候选人 FAQ 知识库在宏佳服务的配置台里(Excel 上传),HR 改一次要登配置台。建议把知识库搬进飞书——用一张多维表格(问题/答案/边界说明三列),我提供接口实时读给宏佳的回复引擎,和岗位标准包同款机制(120秒缓存)。**你改飞书表格,AI 的回答两分钟后就变**,单一真源原则不破。黑客松目录里现成的《秒懂知识库-候选人FAQ.xlsx》《边界说明.md》就是初始素材。
- **沉默跟进**:发出邀约 24/48 小时没回复 → 自动轻追一句;再没回 → 转人工提醒 HR。
- **面试提醒**:面试前一天+前一小时,候选人企微提醒、面试官飞书提醒。日程宏佳已经建了,提醒是顺手的事。
- **到面确认**:面试时间点后 2 小时,问面试官"到面了吗"→ 没到面标记爽约,进流失统计。

### B. 流程推进层(HR 效率)
- **二面三面(详见第五节)**
- **改期/取消(详见第六节)**
- **一面反馈催办**:面试时间过后 24h 反馈字段还空着 → 群里@面试官催填。表里字段都是现成的(一面反馈/二面反馈)。
- **Offer/入职衔接**:走到终面通过 → 提醒 HR 发 offer、办入职,和你已有的转正答辩体系接上。

### C. 数据层(管理视角)
- **漏斗看板**:触达→加友→确认→到面→通过→offer,分岗位分渠道。有了「触达状态」字段,用多维表格仪表盘就能搭,不用写代码。
- **周报自动化**:每周一自动从进度表拉数据生成招聘周报草稿(你有现成的周报体系,挂链接那套规则照用)。

## 五、二面三面怎么整(你点名要的方案)

核心思路:**把"约面"泛化成"约第 N 轮",一套逻辑跑所有轮次。**

表里字段是现成的:一面反馈 / 二面面试官 / 二面时间 / 三面面试官 / 三面时间。

**流程设计(以二面为例):**
1. 面试官把「一面反馈」填成通过 → 服务发现 → 群里@HR:"张三一面通过,请排二面(填二面面试官+二面时间)"
2. HR 填好二面面试官+二面时间 → 服务自检信息齐了 → 调宏佳接口发二面邀约(**老好友直发消息,不再加好友——所以依赖 PR#2**)
3. 候选人确认 → 宏佳建日程 → backfill 回填 → 触达状态=已确认
4. 三面同理,零新逻辑,轮次+1

**要改的东西:**
- 我这边:新规则"轮次推进"(检测 X面反馈=通过 → 提醒;检测 X+1面信息就绪 → 触发),一天工作量
- 宏佳那边:/reach 加个 round 参数(或新接口 /invite),话术模板带轮次("二面约在{time}"),他的任务状态机复用
- 触发是否还要勾框?建议:**首轮勾AI触达=授权 AI 跟这个候选人全程沟通**,后续轮次信息齐了自动发,不用再勾。你拍板。

## 六、改期/取消怎么整

**候选人要改期**(宏佳已能识别 RESCHEDULE 意图):
1. 候选人说"周三不行,周五行吗" → 宏佳提取新时间 → backfill 告诉我
2. 我把表格「触达状态」置为"要改期"+备忘录记候选人想要的时间 → 群里@一面面试官:"张三想改到周五下午,行不行?"
3. 面试官直接改表格里的一面时间(=确认) → 我检测到时间变化 → 通知宏佳 → 候选人收到"已改到周五 14:00"确认消息 → 日程同步更新
面试官不用学任何新操作,改表格就是确认。

**面试官/HR 主动改期**:同上第 3 步,改时间字段就触发候选人通知。这就是第三节卡点 3 的解法,一个规则管双向。

**候选人取消/拒绝**(宏佳已能识别 REJECT):
- 触达状态=已拒绝 → 记流失原因(候选人原话进备忘录) → 不再跟进
- 可选:一个月后自动进"可再捞"名单,换岗位再触达(依赖"回头客"识别)
