# PRODUCT · 句子秒聘表格管理服务（不变内核）

## 目标
把「触达之前」的活自动化：候选人从 AI-HR 表 → 招聘进度表，信息补全、够条件就按铃让触达服务去加人。玄玄负责这半，宏佳负责触达那半。

## 不变内核（改什么都不能破的）
1. **应用身份认证**：只用句子秒聘应用的 app_id/secret（+火山 key）。**绝不依赖任何人的 Claude Code / 个人登录 / 个人电脑**。能上服务器 7×24 自跑，谁离开都不塌。
2. **DRY_RUN 安全**：默认只识别打日志、不写真表。切生产必先演练。
3. **幂等**：同一候选人/记录不重复处理（规则①按候选人ID、规则②按记录ID）。
4. **岗位对齐用豆包**：BOSS 岗位名 → 从进度表现有岗位清单里选（不瞎造），清单现读、改表自动跟上。
5. **简历解析自适应**：自动分 PDF(有文字直接读)/图片·照片(豆包看图)，抽身份/手机等。
6. **触发＝AI触达复选框**（不是一面时间）。人机协作，HR 勾选才触达。
7. **面试官＝谁点约面**（读 last_modified_by，读写同一应用 open_id 才一致）。
8. **数据在飞书表**，服务只搬运+触发；触达对话归宏佳的触达服务。

## 边界（不做的）
- 不做加好友/和候选人对话——那是宏佳触达服务（调它的 HTTP /reach）。
- 不做独立中台界面——HR 操作都在飞书表上。
