# 全自动逐字稿出面评 · 上线清单（方案 A：用户身份）

> 目标：面试结束后，系统**自动**把逐字稿收进【逐字稿链接】→ 自动出一面面评，全程零人工。
> 为什么必须这样：飞书把应用（机器人）当"非组织成员"，"组织内可见"只对**人**开放、不对 app 开放——app 身份读不了别人的妙记（实测 `2091005`）。只有**用户身份**（组织成员）能搜+读组织内可见的妙记。已用真实逐字稿验证过。

## 原理（一句话）

面试用系统建的会自动录制 → 妙记生成「文字记录:线上面试-岗位-姓名」文档 → 服务用**一个服务账号的身份**按标题搜到它、读逐字稿 → 写进【逐字稿链接】→ 触发已有的规则⑨出面评。

## 上线要做的三件事

### 1）在飞书开放平台给 app 开【用户身份】权限（一次，秒聘 owner 本人做）

给用来做 OAuth 的 app（可用秒聘 `cli_aad38`，或另建一个专用 spider app）在**"权限管理"**开这几个（用户身份可用）：

- `docx:document:readonly`（读 docx 正文——raw_content 接口要这个；只给 docs:document.content:read 读会报 99991679，2026-07-23 实测）
- `docs:document.content:read`（读云文档内容）
- `search:docs:read`（搜云文档）
- `offline_access`（拿 refresh_token，token 长期自动刷新；不加则 access_token 约 2 小时过期、无法续）

并在**"安全设置 → 重定向 URL"**加一个回调地址（拿授权码用）：`http://localhost:8080/callback`。
（⚠️别用 `api-explorer/loopback`——会被调试台吞掉 code；localhost 页面打不开，但 code 会停在浏览器地址栏里，复制即可。）

> 若用独立 spider app，把它的 app_id/secret 配成服务器 `.env` 的 `FEISHU_USER_APP_ID`/`FEISHU_USER_APP_SECRET`；用秒聘同一个 app 就不用配，代码自动回退到 `FEISHU_APP_ID/SECRET`。

### 2）超管/你：让面试妙记默认"组织内可见"（一次，对未来所有面试生效）

否则服务账号看不到别人的面试妙记。**不是一篇篇改**——要设成默认。两种方式：

- **面试官各自设**（人少最快）：面试官在自己妙记右上角「分享 → 设置」把可见范围设成「组织内所有人可查看」并设为默认，以后新妙记自动沿用。
- **超管统一设**（要全组织强制）：飞书管理后台 → 产品设置 → 视频会议/妙记 相关的默认可见范围（此开关是否存在待在后台确认）。

### 3）定一个服务账号，做一次 OAuth 授权

- **不能用个人登录**（违玄玄"不依赖任何人电脑/登录"铁律）——用一个 HR/服务账号。
- 该账号要能看到所有面试妙记（即第 2 步生效后）。
- 在服务器跑 `python oauth_bootstrap.py`：打印授权链接 → 服务账号浏览器登录同意 → 粘回 `code` → 令牌落盘 `data/user_token.json`。之后自动刷新，不用再管。

## 上线联调

1. 三件事就绪后，先 `DRY_RUN=true` 跑一轮，看规则⑩日志有没有正确搜到某场面试的文字记录（`[DRY] 规则⑩ 姓名 → 找到文字记录 …`）。
2. 把规则⑩接进 `main.py` 的 `cycle_slow`（在规则⑨**之前**跑，这样同一轮就能"填链接→出面评"）。
3. `DRY_RUN=false`，用一场真实面试端到端验证。
4. ⚠️ **seeding 已内置**：首启把已发生过的面试全视为已处理，绝不回填历史（防刷屏，踩过两次）。

## 相关文件

- `feishu_user.py` — 用户身份客户端（OAuth 换/刷 token + 搜文字记录 + 读正文）
- `oauth_bootstrap.py` — 一次性授权脚本
- `rules.py` `rule10_collect_transcript` — 收集规则（已写，未接主循环）
- 兜底（全自动没上线前）：`面试官-逐字稿SOP.md`（面试官手动贴，但注意导出文档需分享给读取身份）
